# Takuma Saitou

*Updated 2026-04-30*

🛰️ I run a self-bootstrapping multi-agent system as my primary development environment. The agents build the system that orchestrates them.

Nearly all of my token consumption goes into a single piece of software, and that software writes most of itself. Public release is gated on sustained throughput rather than a date — it ships when daily consumption stabilizes around 30B tokens.

[![Tokscale Stats](https://tokscale.ai/api/embed/takuma-saitou/svg)](https://tokscale.ai/u/takuma-saitou)

## 🤖 What's running

Eleven agents, continuously: one coordinator, two planners, four executors, four reviewers. Each role can run on Claude, Codex, Gemini, Qwen, or Kimi — runtimes are swappable per role. Sustained throughput is around 9B tokens per day, with peaks above 14B.

Historical growth has been roughly **5× per month**. The forward target is **3× per month** — deliberately conservative as the system scales. The codebase reached approximately **500K lines** over 75 days, the majority generated by the system itself. The stack began as pure Python; a Rust performance layer landed in April for terminal emulation, token tracking, and the API client.

## 🎯 Where it's going

The aim is **1T tokens per day before year end**. Current focus areas are cross-cluster federation, the TUI, runtime abstraction, networking, and multi-project orchestration. Public release follows once the 30B-tokens-per-day mark is sustained.
