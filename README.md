# Takuma Saitou

*Updated 2026-04-30*

🛰️ I run a self-bootstrapping multi-agent system as my primary development environment. The agents build the system that orchestrates them. The entire project is built solo.

Nearly all of my token consumption goes into a single piece of software, and that software writes most of itself. Public release is gated on sustained throughput rather than a date — it ships once daily consumption stabilizes in the 20–30B token range.

[![Tokscale Stats](https://tokscale.ai/api/embed/takuma-saitou/svg)](https://tokscale.ai/u/takuma-saitou)

## 🤖 What's running

Eleven agents, continuously: one coordinator, two planners, four executors, four reviewers. Each role can run on Claude, Codex, Gemini, Qwen, or Kimi — runtimes are swappable per role. Sustained throughput is around 9B tokens per day, with peaks above 14B.

## 🧬 What it does

The system is a single integrated platform rather than a collection of scripts.

Tasks are first-class objects with structured lifecycles: planning, acceptance criteria, multi-reviewer verdicts with fix criteria, post-merge verification, and dependency tracking. Tasks branch into kinds — bug, code, hotfix, planning — each with its own workflow.

Knowledge is shared through versioned documents, subscribed topics, mention-aware messaging, and per-agent memory. Coordination happens through structured talk, lifecycle-aware nudges, and instance-aware sessions where one agent template can fan out to multiple runtimes.

Around all of that sits a hook system (four trigger types × ten action types), an error pipeline that ingests, fingerprints, correlates, and auto-routes failures back into the task queue, and a token observability layer with live TUI. Operations include blue-green deployment with atomic version swaps, transport-agnostic bundle distribution, multi-project orchestration, and a standalone watchdog that survives server crashes.

Across the 75 days: **2,085 tasks completed**, **9,180 commits**, with the verification pipeline rejecting roughly **47% of review submissions** — nearly half of agent output gets pushed back on the first pass.

Historical growth has been roughly **5× per month**. The forward target is **3× per month** — deliberately conservative as the system scales. The codebase reached approximately **500K lines** over 75 days, the majority generated by the system itself.

The stack started as pure Python and grew layered. A Rust core arrived in April — seven crates covering an in-house terminal emulator, token usage tracking, and a typed HTTP/WebSocket client. The web frontend is React/TypeScript. All three languages share generated code from a single OpenAPI schema. Quality is enforced by **over 200 project-specific gate checks** spread across roughly fifty checker scripts, plus eighty database migrations applied incrementally across the 75 days.

## 🎯 Where it's going

My personal ceiling sits at a sustained **20–30B tokens per day** — the threshold I've set for going public. The longer-term **1T tokens per day before year end** is a project-level milestone: once released, throughput is the combined output of the system itself and everyone running it. Current focus areas leading up to release are cross-cluster federation, the TUI, runtime abstraction, networking, and multi-project orchestration.
